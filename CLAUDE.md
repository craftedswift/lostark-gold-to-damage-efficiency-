# Working on this repo

**Always run `git pull` before starting any work in a new session** (a new
day, or picking this project back up after a break) — this repo has more
than one person working on it, and starting from a stale local `main` risks
silently working on top of outdated research/code or creating avoidable
merge conflicts. Do this before reading files or editing anything, not just
before committing.

See [README.md](README.md) for project orientation and
[RESEARCH.md](RESEARCH.md) for the actual project handoff doc — read
RESEARCH.md before proposing any code.
